# 快速测试指南 / Quick Test Guide

## 🚀 快速开始

### 1. 编译项目
```bash
cd MyBuman/GameController3
./build_with_pause_button.sh
```

### 2. 运行 GameController
```bash
cargo run --release
```

## 🧪 测试步骤

### 测试 1: 验证按钮显示
1. 启动 GameController
2. 进入主界面
3. **检查点**：
   - ✅ 顶部应该有一个黄色的"⏸️ 全部暂停 / PAUSE ALL"按钮
   - ✅ 按钮占据整行，非常显眼
   - ✅ 按钮文字清晰可读

### 测试 2: 测试按钮可用性
1. 在不同游戏状态下观察按钮：
   - **Initial 状态**：按钮应该可用（黄色）
   - **Playing 状态**：按钮应该可用（黄色）
   - **Timeout 状态**：按钮应该可用（黄色）
   - **Finished 状态**：按钮应该禁用（灰色）

### 测试 3: 测试暂停功能
1. 开始一场比赛（点击 Ready → Set → Playing）
2. 点击"⏸️ 全部暂停 / PAUSE ALL"按钮
3. **检查点**：
   - ✅ 游戏状态应该变为 "Timeout"
   - ✅ 所有机器人应该停止移动
   - ✅ 计时器应该停止
   - ✅ 界面显示暂停状态

### 测试 4: 测试恢复功能
1. 在暂停状态下
2. 点击"Ready"或"Standby"按钮
3. **检查点**：
   - ✅ 游戏状态应该恢复
   - ✅ 可以继续比赛流程

### 测试 5: 测试悬停效果
1. 将鼠标悬停在"全部暂停"按钮上
2. **检查点**：
   - ✅ 按钮颜色应该变深
   - ✅ 阴影效果应该增强
   - ✅ 应该显示工具提示

### 测试 6: 测试禁用状态
1. 在比赛结束后（Finished 状态）
2. **检查点**：
   - ✅ 按钮应该变为灰色
   - ✅ 鼠标悬停时显示禁止符号
   - ✅ 点击无效果

## 🐛 常见问题

### 问题 1: 按钮不显示
**解决方案**：
```bash
# 重新编译前端
cd frontend
npm run build
cd ..
cargo build --release
```

### 问题 2: 按钮样式不正确
**解决方案**：
- 检查 Tailwind CSS 是否正确编译
- 清除浏览器缓存
- 重新编译前端

### 问题 3: 点击按钮无反应
**解决方案**：
- 检查浏览器控制台是否有错误
- 确认后端正在运行
- 检查 action 是否正确传递
